• The website uses 2 API's: Google's Geocoder API and Google's Timzone API. The Geocoder returns a pair of coördinates based on the user input. This input can be a city, land, continent, anything. These coördinates are then offered to the Timezone API. This returns a time offset with respect to the clients time. This means the current timezone of the client must also be known. Luckuliy there is a JS funtion to find these. Equating the time offset with the clients time will yeild the local time at the requested location.
  • I came up with this idea because my brother is currently residing in Australia. In his current location, we have a 7 hour time difference. Autralia is divided in no less than 5 timezones: +8, +9,5, +10 hours with respect to GMT, and 2 timezones that are +9,5/+10,5 and +10/+11 with respect to GMT, depending on the time of the year. This can be confusing, but using this website it is easy to find the current time at any location in australia (and anywhere else on the world)

TimeZone finder

Welcome to my Timezone finder. To find the time in a specific place, enter the name of this place in the form and press "geocode". The clock will start running at the local time of that place.
